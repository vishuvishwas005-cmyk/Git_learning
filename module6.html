Module 6: Git Best Practices

Commit Best Practices

  • Write clear, descriptive commit messages
  • Keep commits focused on single changes
  • Use present tense: "Add feature" not "Added feature"
  • Reference issues: "Fix #123: resolve login bug"

Branching Strategy

Git Flow:

  • main: Production-ready code
  • develop: Integration branch
  • feature/*: New features
  • hotfix/*: Critical fixes
  • release/*: Release preparation

Pull Request Guidelines

  • Provide clear descriptions
  • Reference related issues
  • Keep PRs small and focused
  • Request reviews from appropriate team members
  • Test thoroughly before submitting

Repository Organization

  • Use .gitignore to exclude unnecessary files
  • Include README.md with project description
  • Use CONTRIBUTING.md for contribution guidelines
  • Maintain a CHANGELOG.md for releases

Collaboration Tips

  • Communicate with your team
  • Don't force push to shared branches
  • Keep branches up to date with main
  • Use code reviews effectively
  • Resolve conflicts promptly

Common Mistakes to Avoid

  • Committing large binary files
  • Using generic commit messages like "fix"
  • Working directly on main branch
  • Not pulling before pushing
  • Ignoring merge conflicts

Useful Git Aliases

git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status

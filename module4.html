Module 4: Working with Remotes

What are Remote Repositories?

Remote repositories are versions of your project hosted on the internet (GitHub, GitLab, Bitbucket). They enable collaboration and backup.

Adding a Remote

Connect your local repo to a remote:

git remote add origin https://github.com/username/repo-name.git

List remotes:

git remote -v

Cloning a Repository

Download an existing repository:

git clone https://github.com/username/repo-name.git

This creates a local copy with remote tracking.

Pushing Changes

Upload your commits to the remote:

git push origin main

Push a new branch:

git push -u origin feature-branch

Pulling Changes

Download and merge remote changes:

git pull origin main

Fetch without merging:

git fetch origin

Forking and Pull Requests

  1. Fork the repository on GitHub
  2. Clone your fork: git clone https://github.com/your-username/repo-name.git
  3. Create a feature branch
  4. Make changes and push
  5. Create a Pull Request on GitHub

Common Remote Commands

  • git remote rename origin upstream - Rename remote
  • git remote remove origin - Remove remote
  • git push origin --delete branch-name - Delete remote branch

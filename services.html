Git Status: Track Your Repository State

What is Git Status?

The git status command shows the current state of your working directory and staging area. It's one of the most frequently used Git commands.

Basic Usage

git status

This command displays:

  • Current branch
  • Untracked files
  • Modified files
  • Staged files
  • Status compared to remote

Understanding the Output

Untracked Files

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        newfile.txt

These are new files Git doesn't know about yet.

Modified Files

Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
        modified:   index.html

These files have been changed but not staged.

Staged Files

Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        modified:   index.html

These changes are ready to be committed.

Advanced Options

  • git status --short or git status -s: Concise output
  • git status --porcelain: Machine-readable format
  • git status --ignored: Show ignored files
  • git status --ahead-behind: Show ahead/behind remote

Practical Examples

Short Status

$ git status -s
 M index.html
?? newfile.txt
 A staged.txt

Legend: M=modified, A=added, ??=untracked

After Adding Files

$ git add .
$ git status
On branch dev
Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        new file:   newfile.txt
        modified:   index.html

Best Practices

  • Run git status frequently to stay aware of your repository state
  • Use git status -s for quick checks
  • Always check status before committing
  • Understand the difference between staged and unstaged changes

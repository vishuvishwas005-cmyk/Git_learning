Module 3: Branching & Merging

What are Branches?

Branches allow you to work on different features or fixes simultaneously without affecting the main codebase.

  • main/master: The default branch
  • feature branches: For new features
  • bugfix branches: For fixing bugs

Creating and Switching Branches

Create a new branch:

git branch feature-name

Switch to a branch:

git checkout feature-name

Create and switch in one command:

git checkout -b feature-name

Listing Branches

See all branches:

git branch

See all branches including remote:

git branch -a

Merging Branches

Merge a feature branch into main:

git checkout main
git merge feature-name

Types of merges:

  • Fast-forward: When no commits on main since branching
  • Three-way merge: Creates a merge commit

Handling Merge Conflicts

When files conflict:

  1. Git will mark conflicts in files
  2. Edit files to resolve conflicts
  3. Stage resolved files: git add file.txt
  4. Complete merge: git commit

Advanced Merging Concepts

Merge Strategies

  • Recursive: Default strategy for merging two branches
  • Resolve: Uses three-way merge for simple cases
  • Ours: Keeps current branch changes, discards others
  • Theirs: Keeps other branch changes, discards current

Merge vs Rebase

Merge: Creates a merge commit, preserves history

git merge feature-branch

Rebase: Replays commits on top of another branch

git rebase main

Practical Merge Workflow

  1. Create feature branch: git checkout -b feature/new-feature
  2. Make changes and commit: git add . && git commit -m "Add feature"
  3. Switch to main: git checkout main
  4. Pull latest changes: git pull origin main
  5. Merge feature: git merge feature/new-feature
  6. Push merged changes: git push origin main
  7. Clean up: git branch -d feature/new-feature

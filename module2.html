Module 2: Basic Git Commands

Initializing a Repository

Create a new Git repository in your project folder:

git init

This creates a hidden .git folder that tracks your files.

Checking Repository Status

See the current state of your files:

git status

This shows untracked, modified, and staged files.

Adding Files to Staging

Stage specific files:

git add filename.txt

Stage all files:

git add .

Committing Changes

Save your staged changes:

git commit -m "Your descriptive commit message"

Good commit messages are clear and concise.

Viewing Commit History

See all commits:

git log

View a compact log:

git log --oneline

Practical Exercise

  1. Create a new folder and navigate to it
  2. Run git init
  3. Create a file (e.g., README.md)
  4. Check git status
  5. Add the file with git add .
  6. Commit with git commit -m "Add README"
  7. View history with git log
